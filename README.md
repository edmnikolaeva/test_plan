**Domain:**
- Online hotel booking
- Short-stay capsule hotels
- Web self-service platform

**Scope:**
- User registration
- Personal account
- Hotel booking flows
- Booking management
- Interactive map

**Business Value**  
- Prevent user drop-off after registration / booking  
- Avoid support overload from broken flows  
- Reduce post-launch hotfixes
  
**Goal:**
- Verify readiness of version 2.2 for release
- Validate core user journeys
- Detect blocking defects before launch

**Key Pain Points:**
- Map interaction is unreliable
- Email notifications are not delivered
- Guest booking confirmation missing
- Email cannot be edited in profile
- Booking cancellation is blocked

**Solution:**
- Extend and execute test plan
- Validate critical user flows
- Compare expected vs actual behavior
- Identify failed scenarios
- Highlight product risks

**Business Value:**
- Early detection of release blockers
- Reduced risk of failed bookings
- Improved user experience at launch
- Better visibility of product risks
- Support for release decision
